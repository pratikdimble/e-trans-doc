Overview

The E-Trans Admin system is a comprehensive logistics and transport management platform. It manages companies, customers, vehicles, routes, materials, trips, invoices, credit notes, and provides reporting and analytics.

Modules

Company Module

  • Create, edit, and delete company details
  • Store company GSTIN, PAN, and address
  • Link to customers, vehicles, trips, and bills

Customer Module

  • Maintain customer master data
  • Supports GSTIN and non-GST customers
  • Used in trip billing and GSTR reports

Vehicle Module

  • Register and manage vehicles
  • Track trips and revenue per vehicle
  • Integrates with dashboard vehicle revenue chart

Routes Module

  • Define transport routes
  • Associate trips with source and destination
  • Calculate route-based billing

Materials Module

  • Maintain material master data
  • Track quantities per trip
  • Used in HSN reporting in GSTR-1

Trips Module

  • Log trips with vehicle, customer, and material details
  • Track trip charges and GST for reporting
  • Trip data feeds into bills and dashboard summaries

Bill Module

  • Create invoices for trips
  • Track total amount, received, and balance
  • Supports B2B (GSTIN) and B2C (No GST) differentiation

Credit Note Module

  • Issue credit notes for billing adjustments
  • Supports customer-wise aggregation
  • Integrated into GSTR-1 reporting and totals

Dashboard Summary Module

  • Filters: Year, Month, Quarter, Start/End Date, Search (Customer/Vehicle/Bill No)
  • Summary Cards: Total Revenue, Total Received, Total Pending
  • Charts: Top 5 Customer Revenue, Monthly Trend, Pending vs Received, Vehicle Revenue
  • Paginated Invoices Table

GSTR-1 Report Module

  • Filters: Customer, From Date, To Date
  • Summary Cards: Taxable, CGST, SGST, IGST
  • Customer-wise aggregation by GSTIN
  • B2B and B2C invoice segregation
  • HSN summary (material-wise quantity)

Authentication & Refresh Flow

  • Axios instance automatically adds Authorization: Bearer <token>
  • On 401 errors, refresh token is used to request a new access token
  • Requests are queued while refreshing
  • User is logged out if refresh fails
